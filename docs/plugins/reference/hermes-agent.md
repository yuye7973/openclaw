`hermes deploy-plan` creates a dry-run role deployment plan for sequential
Hermes planner, worker coder, verifier, and memory summarizer handoff. It does
not execute tasks or start parallel model runs.
`--taskflow-skeleton` previews the managed Task Flow shape without writing to
the Task Flow registry.
`--write-taskflow-skeleton` persists that dry-run skeleton as a workspace report
artifact while still leaving the Task Flow registry untouched.
`validate-taskflow-skeleton` checks the persisted skeleton's steps, gates,
report requirements, resource policy, and high-risk action blocks before any
future registry promotion.
`stage-taskflow-registry` writes a dry-run registry staging artifact only after
that validation passes. It leaves the real Task Flow registry untouched and
keeps worker execution and model invocation disabled.
`gate-taskflow-registry` writes the explicit promotion gate report for that staging artifact. It rechecks the source skeleton, keeps `registry_write_allowed` false, and records that manual approval is required before any real registry write path exists.
`write-taskflow-registry-noop` writes the no-op registry writer adapter artifact. It proves the adapter boundary while keeping the real SQLite Task Flow registry untouched.
`preview-taskflow-registry-diff` writes the read-only preview diff artifact from that no-op write record. It previews the create-or-update flow operation without registry read/write access.
`request-taskflow-registry-approval` writes the pending operator approval artifact for that preview while keeping actual approval, registry writes, SQLite writes, worker execution, and model invocation disabled.
`validate-taskflow-registry-approval` writes the approval validation artifact and still keeps registry writes, SQLite writes, worker execution, and model invocation disabled.
`snapshot-taskflow-registry-dry-run` normalizes an exported `openclaw tasks flow list --json` artifact into a read-only registry snapshot. `read-taskflow-registry-dry-run` writes the read adapter report from that optional snapshot artifact and keeps SQLite reads/writes, registry writes, worker execution, and model invocation disabled. `review-taskflow-registry-write` writes a review-only writer gate for the proposed create or update operation while keeping all real writes disabled. `apply-taskflow-registry-noop` writes the final no-op apply artifact for that reviewed operation without registry writes, SQLite writes, worker execution, or model invocation. `validate-taskflow-registry-approval-token` validates the explicit operator token challenge and stores only a fingerprint while still keeping all real write and execution flags disabled. `write-taskflow-registry-writer-contract-fixture` writes the dry-run writer contract fixture with registry and SQLite commits disabled. `report-taskflow-registry-contract-coverage` writes the read-only contract coverage report and checks required fields plus safety flags before any real writer exists. `preflight-taskflow-registry-promotion` writes the read-only promotion preflight report and keeps every real writer, SQLite, worker, and model path disabled. `write-taskflow-registry-rollback-contract-fixture` writes the dry-run rollback contract fixture and still keeps every commit path disabled. `report-taskflow-registry-rollback-contract-coverage` writes the read-only rollback coverage report for required fields, manual rollback trigger, operator review, disabled commits, and disabled execution paths. `summarize-taskflow-registry-final-writer-readiness` writes the final read-only summary and requires writer contract coverage, promotion preflight, and rollback coverage to be green before any real writer plan can be considered. `plan-taskflow-registry-real-writer-implementation` writes a dry-run implementation plan and guardrail checklist only; it keeps real writer creation, registry writes, SQLite writes, worker execution, and model invocation disabled. `report-taskflow-registry-real-writer-implementation-plan-coverage` writes the read-only coverage report for that plan and checks implementation phases, guardrails, required validation, rollback path, and disabled flags. `write-taskflow-registry-real-writer-api-contract-fixture` writes a dry-run API contract fixture only; it defines input/output schemas, approval-token boundary, rollback precondition, and disabled commit strategy without creating writer code. `report-taskflow-registry-real-writer-api-contract-coverage` writes the read-only API contract coverage report and proves those contract fields plus disabled execution flags stay locked before any writer code exists. `preflight-taskflow-registry-real-writer-promotion` writes the read-only promotion preflight report and confirms API contract coverage green, approval boundary locked false, rollback precondition present, disabled commit strategy locked, and all write/execution paths disabled. `report-taskflow-registry-real-writer-promotion-preflight-coverage` writes the read-only preflight coverage report and rechecks that the preflight green state stays complete without creating writer code. `report-taskflow-registry-real-writer-final-readiness-lock` writes the read-only final readiness lock report and keeps the real writer path disabled until a separate explicit operator approval boundary exists. `write-taskflow-registry-real-writer-activation-approval-token-fixture` writes a dry-run challenge/fingerprint fixture only; the fingerprint is not operator approval and all real write/execution paths stay disabled. `report-taskflow-registry-real-writer-activation-approval-token-coverage` writes the read-only coverage report that proves those approval-token boundaries remain locked. `preflight-taskflow-registry-real-writer-activation-approval` writes the read-only activation approval preflight report and rechecks coverage green, fixture-not-approval, raw-token non-persistence, rollback precondition, disabled commit strategy, and disabled write/execution flags. `report-taskflow-registry-real-writer-activation-approval-preflight-coverage` writes the read-only coverage report for that preflight and keeps the real writer, registry, SQLite, worker, and model paths disabled. `report-taskflow-registry-real-writer-activation-final-readiness-lock` writes the final read-only activation readiness lock and still does not create a writer or invoke registry, SQLite, worker, or model paths. `report-taskflow-registry-real-writer-activation-final-readiness-lock-coverage` writes the read-only coverage report for that lock and proves the lock green state, approval preflight coverage, token coverage, rollback precondition, disabled commit strategy, and disabled write/execution flags remain complete. `report-taskflow-registry-real-writer-activation-release-checklist` writes the read-only release checklist report and rechecks final readiness lock coverage, activation lock, approval preflight coverage, token coverage, rollback, disabled commit strategy, and disabled execution flags before any writer exists. `report-taskflow-registry-real-writer-activation-release-checklist-coverage` writes the read-only coverage report for that checklist and keeps the real writer, registry, SQLite, worker, and model paths disabled. `write-taskflow-registry-real-writer-activation-operator-handoff-packet` enable registry, SQLite, worker, or model paths. `report-taskflow-registry-real-writer-activation-final-safety-summary` writes the read-only final safety summary and keeps release checklist coverage, safe task proposer or writer work. `write-taskflow-registry-next-safe-task-proposer-fixture` writes a dry-run deterministic next-task proposal fixture that may reference a Hermes model-decision executor loop while keeping `execution_started: false`. `write-taskflow-registry-task-dialoguer-reasoning-policy` writes read-only reasoning rules for goal understanding, risk classification,
