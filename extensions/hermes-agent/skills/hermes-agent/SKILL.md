# Hermes Agent

Use this skill when a request should be converted into a bounded OpenClaw task
package before execution.

## Rules

- Stay inside the selected OpenClaw workspace.
- Produce a dry-run task package before execution.
- Keep high-risk actions behind a human approval request.
- Never run real trading, payment, credential, deployment, or external-write
  actions from this skill.
- Return changed files, validation results, risk, rollback path, and next safe
  task.

## Output Contract

The runtime returns:

- `task_package`: bounded execution plan.
- `envelope`: AI-to-AI review message.
- `audit`: dry-run audit record.
- `approval_request`: present only for high-risk actions.
- `model_decision`: model-backed Hermes/OpenClaw dialogue decision when using `hermes decide`; with `--output-dir`, it is persisted as a readback artifact with `model_invoked: true`.
- `model_decision_coverage_report`: read-only coverage report when using `hermes report-model-backed-dialogue-decision-coverage`; it proves the persisted decision invoked a model, parsed JSON, selected provider/model, includes validation commands and disabled registry/SQLite/worker/live-action notes, while the report itself keeps `model_invoked: false`.
- `model_decision_handoff_plan`: read-only/dry-run handoff plan when using `hermes plan-model-backed-dialogue-decision-handoff`; it can only pass the covered model dialogue forward as controlled executor intake context and keeps operator approval, registry writes, SQLite writes, worker execution, model invocation, and live actions disabled.
- `model_decision_handoff_intake_coverage_report`: read-only coverage report when using `hermes report-model-backed-dialogue-handoff-intake-coverage`; it proves the handoff is ready only as controlled executor intake context and keeps operator approval, execution, registry writes, SQLite writes, worker execution, model invocation, and live actions disabled.
- `model_decision_intake_release_summary_report`: read-only release summary when using `hermes report-model-backed-dialogue-intake-release-summary`; it consumes green handoff intake coverage and keeps the result as controlled executor intake context only, with operator approval, execution, registry writes, SQLite writes, worker execution, model invocation, and live actions disabled.
- `model_decision_intake_release_summary_coverage_report`: read-only coverage report when using `hermes report-model-backed-dialogue-intake-release-summary-coverage`; it proves the release summary is ready, intake coverage is green, the controlled executor intake-context boundary is complete, and operator approval, execution, registry writes, SQLite writes, worker execution, model invocation, and live actions remain disabled.
- `model_decision_intake_activation_preflight_report`: read-only preflight report when using `hermes preflight-model-backed-dialogue-controlled-executor-intake-activation`; it consumes release-summary coverage, proves the controlled executor intake activation boundary remains locked, and keeps operator approval, execution, registry writes, SQLite writes, worker execution, model invocation, and live actions disabled.
- `model_decision_intake_activation_preflight_coverage_report`: read-only coverage report when using `hermes report-model-backed-dialogue-controlled-executor-intake-activation-preflight-coverage`; it consumes intake activation preflight, proves the preflight is green with release-summary coverage and intake-context boundaries intact, and keeps operator approval, execution, registry writes, SQLite writes, worker execution, model invocation, and live actions disabled.
